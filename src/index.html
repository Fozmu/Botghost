Ticket Transcript

Are you looking for a free ticket transcript? We have a step-by-step guide listed below:


  • Head over to Cookie API
  • Create an account and head over to Overview. Once there copy the Api Key.
  • Go back to Botghost and add an API request.

  • Set the method as Post and as the url enter https://api.cookie-api.com/api/transcript
  • In the url params add channel_id as the key and enter a valid channel id in the value section.

    • (This can be replaced by a variable after step 10.)

  • Head over to the HTTP Headers and add Authorization as your key.

    • In the value section enter the API Key from before.

  • Head over to the Request Body. Add bot_token as the key.
  • For the value we will need the bot token. {TOKEN_SECRET} wont work. we will need to use our real bot token.

    • (Bot token can be reset in the Discord developer page under the bot section.)


  • Test the request if you get a code 400 check for any typos and make sure you dont have any variables.

    • If you get a code 200 everything worked.

  • copy the variable for the url that you received.
  • Replace all ids with variables so you can use it in your system.

You should be all done, if you have any questions ask a CS member.

  • Cookie's API Docs

Gemini ChatBot

To make it work, You must get a Gemini API key at Google AI Studio.


Required Blocks:

  • Option Text
  • Set a Unique Variable 3x
  • Send an API Request
  • Comparison Condition
  • Send message block 2x

Step-by-Step explanation

  1. Set text option and name it option_question

  2. Set a Unique Variable

    3. Variable Value

    gemini_api_key Your API Key


  3. Set a Unique Variable

    4. Variable Value

    language English


  4. Set a Unique Variable

    5. Variable Value

    system_instruction You are a Chatbot.


  5. Send an API Request

    6. Name

    Gemini

    Method

    POST


    Url: 
    https://generativelanguage.googleapis.com/v1beta/models/gemini-2.0-flash:generateContent

    URL Params

    Key Value

    key {gemini_api_key}


    HTTP Headers

    Key Value

    Content-Type application/json


    Request Body

    Key Value 

    system_instruction  {     "parts":       { "text": "Response in {language}.{system_instruction}"}}
    contents            [     {       "parts": [         {           "text": "{option_question}"         }       ]     }   ]

    tools               [     {         "googleSearch":{}         }     ]
  6. Comparison Condition

    7. Base Value Comparison Type

    {gemini.status} Equal to


    New Condition {gemini.status} must be equal to 200

  7. If equal to 200 then paste {gemini.response.candidates.0.content.parts.0.text} into a send a message block.

    8. {gemini.response.candidates.0.content.parts.0.text} is the response from Gemini.


  8. Save the Command

GoDaddy Domain Issue

If you experiencing a problem with GoDaddy Domain and BotPanel. This Documentation is for you.


To use a Custom Domain on BotPanel you must connect it with your Domain Host. Sadly the GoDaddy Domain Service doesn't allow us to place a @ in the CNAME record.


CNAME @ hosting.botpanel.gg Auto/Default On GoDaddy putting the @ as the Hostname is impossible. This is why you need to change the Nameservers.


You can use the Nameservers listed below:

  • CloudFlare
  • Netlify

Delete all DNS Records in GoDaddy rename the Nameservers in GoDaddy to the
Nameserver Provider (Netlify or CloudFlare) create new DNS Records in Netlify or CloudFlare
and that should be it.


  • CloudFlare Docs
  • Netlify Docs
